## Deviations from the previous handoff's plan, and why

1. **Added `--enforce-eager`, raised `--gpu-memory-utilization` 0.90 to
   0.92.** The suggested args crashed engine init with "No available
   memory for the cache blocks" (Available KV cache memory: -2.05 GiB).
   Measured on this 24 GB 4090: weights 16.95 GiB plus roughly 6.7 GiB of
   CUDA-graph, torch.compile, and activation overhead, which does not fit
   even at util 1.0. `--enforce-eager` drops the CUDA-graph overhead and
   leaves a healthy 2.51 GiB for the KV cache. Inference is somewhat
   slower without CUDA graphs; that is acceptable because perf
   optimization (including the TRT-LLM path) is a later phase.

2. **Brainstem runs as a plain-Python uvicorn process, not the Docker
   compose service.** The `docker compose build` for the brainstem image
   ran past 20 minutes on the `pip install sentence-transformers` step
   (the torch CUDA wheel is slow to install on WSL2 overlayfs) and was not
   finishing. Pivoted to `python -m uvicorn brainstem_4070.server:app`
   with only the light dependencies installed. The lazy-import change
   below is what makes that possible. The Docker path is still valid and
   can be revisited; the milestone is the 4070 to 4090 link, not the
   container packaging.

## Environmental gotchas discovered

- **Stray WSL vLLM on the 4090.** A WSL-native `vllm serve` process was
  holding port 8000 serving Qwen3-0.6B. It is not mentioned in any
  handoff. Stopped it gracefully so the Docker container could own 8000.
- **Docker credential helper fails in non-interactive sessions on the
  4070.** `docker build` and `docker pull` over SSH fail with "A
  specified logon session does not exist" because
  `docker-credential-desktop` cannot reach DPAPI without an interactive
  logon. Workaround: point `DOCKER_CONFIG` at a directory whose
  `config.json` is `{"credsStore":"","auths":{}}`. Anonymous pulls of
  public images then work. That directory is at `~/nexus_docker_cfg` on
  the 4070.
- **Processes started over SSH die when the session closes.** Windows
  OpenSSH kills the child process tree on disconnect, and `Start-Process`
  children die with it. Use `Invoke-CimMethod Win32_Process Create` to
  launch anything that must outlive the SSH session. The brainstem
  uvicorn worker and the HF model downloads were launched that way.
- The brainstem service is currently started by an ad-hoc worker script
  at `~/nexus_brainstem_run_worker.ps1` on the 4070 (it sets PYTHONPATH to
  the repo root plus `nodes/`, and logs to `~/nexus_brainstem_run.log`).
  That is not yet a tracked, reproducible launcher. See next steps.

1. Make the brainstem launch reproducible: either a tracked launcher
   script, or fix the Docker build. Installing the CPU-only torch wheel
   (`--index-url https://download.pytorch.org/whl/cpu`) before
   sentence-transformers would cut the build time massively, since the
   brainstem runs embeddings on CPU anyway.
2. Bring up the NAS memory service on the 4070 so `/embed` and
   `/stm/write` work end to end (Chroma on the 4070's local SSD).

   already emit tool calls (`--enable-auto-tool-choice --tool-call-parser
   hermes` are set); the 4070 needs a tool endpoint the model can call
   back into.

   plain orchestration logic is not enough for the MVP.
5. Start Phase 0 Component G, the metric harness, per the Experiment 1
   design. It lands first because every later component's rolling numbers
   depend on it.

## Open questions

- Containerize the brainstem (fix the slow Docker build) or keep it as a

  the repo's deployment model is docker-compose.
- The 4090's vLLM is running `--enforce-eager`. When inference speed
  starts to matter, revisit: either accept a smaller `--max-model-len` so
  CUDA graphs fit, or move to the TRT-LLM path (deferred per the original
